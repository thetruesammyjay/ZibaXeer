Scalability Decisions

Why This Scales

1) Event-Driven Pipeline

Vault and trade events are buffered as jobs, absorbing burst traffic without blocking user flows.

2) Queue-Based Workers

Trade persistence, snapshots, and follower sync are retriable and horizontally scalable.

3) Read/Write Separation

Financial truth on-chain. Analytics served from materialized DB views for stable latency.

Bottleneck Axis Scaling Lever
Vault Count Growth Parallel event listeners + queue partitioning
Follower Growth Worker concurrency + indexed DB writes
Trade Frequency Bursts Queue backpressure + retry/backoff
API Read Load Replicated backend nodes + cached aggregates
