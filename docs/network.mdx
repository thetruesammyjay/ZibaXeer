## Why Paxeer Network

Paxeer's funded wallet model provides immediate capital liquidity for both vault leaders and followers, removing the cold-start problem that plagues most copy-trading protocols.

| Feature | Benefit for ZibaXeer |
|---|---|
| ~2s block time | Trade events indexed in near real-time |
| Instant finality | No reorg risk — indexer never needs to handle chain reorgs |
| EVM-compatible | Full ethers.js / viem / Wagmi compatibility |
| Funded wallets | Leaders and followers start with up to $50,000 USDL |
| IBC-ready | Future cross-chain vault mirroring via IBC |
| Colosseum integration | Leaders ranked by on-chain performance, feeding Argus Oracle |

## Argus Risk Engine

The Argus Risk Engine is Paxeer's native reputation system. ZibaXeer integrates it via the `ArgusOracle` contract.

- Scores are derived from **500+ on-chain signals** across a leader's trading history
- Minimum score required to create a vault: **600 / 1000**
- Scores are queried on-chain by `RiskManager` before every trade
- Lower-scored leaders are blocked from executing trades — protecting follower funds

```solidity
// RiskManager.sol
uint256 public constant MIN_REPUTATION_SCORE = 600;

function validateTrade(address vaultLeader) external view returns (bool) {
    require(!isVaultFrozen[msg.sender], "Vault is frozen");
    bool isEligible = argusOracle.isEligible(vaultLeader, MIN_REPUTATION_SCORE);
    require(isEligible, "Leader reputation below threshold");
    return true;
}
```
